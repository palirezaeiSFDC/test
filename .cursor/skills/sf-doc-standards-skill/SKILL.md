# Documentation Standards — Templates

Paste-ready templates for the three standard document types. Use these templates verbatim and fill in the bracketed fields — do not re-assemble document structure from memory.

| Document type | Template | Used by |
|---|---|---|
| Impact Assessment (7 sections) | [`impact-assessment-template.md`](impact-assessment-template.md) | `/sf-impact-assessment` |
| Solution Design (11 sections) | [`solution-design-template.md`](solution-design-template.md) | `/sf-solution-design` |
| Technical Design (11 sections) | [`technical-design-template.md`](technical-design-template.md) | `/sf-feature-implement` (post-implementation) and standalone use |

## Section Order

### Impact Assessment

1. Executive Summary
2. Scope
3. Affected Components
4. Risk Analysis
5. Testing Scope
6. Deployment Notes
7. References

### Solution Design / Technical Design

1. Problem Statement
2. Assumptions
3. High-Level Design
4. Design Options (Solution Design) / Component Design (Technical Design)
5. Recommendation (Solution Design) / Integration Design (Technical Design)
6. Data Model Changes
7. Integration Points (Solution Design) / Automation / Trigger Framework (Technical Design)
8. Governor Limit Considerations
9. Org Verification Required (Solution Design) / Test Strategy (Technical Design)
10. Implementation Sequence (Solution Design) / Deployment Plan (Technical Design)
11. References
